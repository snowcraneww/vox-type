# Evaluator Rubric

在任务完成后、正式接受前，用这张表评审。总分 10 分，低于 8 分不能标记为 `passing`，除非维护者明确接受风险。

| 维度 | 问题 | 分数 0-2 | 备注 |
| --- | --- | --- | --- |
| 正确性 | 是否完成了当前 feature list 项的目标行为？ |  |  |
| 验证 | 要求的命令或手工检查是否真的运行并记录证据？ |  |  |
| 范围纪律 | 是否保持 WIP=1，没有顺手扩展无关范围？ |  |  |
| 可复现性 | 新会话能否只靠仓库文件恢复上下文？ |  |  |
| 开源安全 | 是否避免写入密钥、私有数据、受版权保护代码或机器专属假设？ |  |  |

## 结论

- Accept：8-10 分，且没有阻塞性缺口。
- Revise：5-7 分，需要补证据或修范围。
- Block：0-4 分，不能继续叠加新工作。

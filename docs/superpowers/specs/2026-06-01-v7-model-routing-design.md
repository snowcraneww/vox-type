# V7 模型选择与配置页重构设计

## 背景

V6 已完成百度短语音识别真实调用，但模型选择页仍残留“本地 / 云端 API / MiniMax / 百度”混合结构。这个结构会让用户误以为只有一个全局模型，且主界面准备状态把“本地识别”和“云端 API”拆成两项，不能回答真实使用问题：按下某个输入模式时到底会用哪个模型。

V7 的核心目标是把模型选择从“全局引擎”改为“按输入模式选择默认模型”。按住说话和连续输入有不同交互语义，后续也会适合不同 ASR 能力：按住说话适合录完后整段识别，连续输入未来更适合实时流式识别。

## 产品目标

- 主界面准备状态只展示真实日常使用所需的系统能力。
- 按住说话和连续输入分别拥有自己的默认转写模型。
- 模型选择页同时承担“选择默认模型”和“配置模型能力”两个职责，但在信息架构上清楚分开。
- 从 V7 用户界面移除 MiniMax 配置入口；历史调研和代码可在后续清理，但不能再作为用户可选模型出现。
- 百度实时 WebSocket ASR 作为 V8 预留模型入口，不在 V7 实现真实流式调用。
- 识别记录展示每条记录的输入方式、模型、录音时长和字数，便于比较模型效果。
- 为识别准确率优化预留确定性路线：百度 `lm_id` 和本地术语纠错表。

## 主界面准备状态

准备状态保持四项，避免三项排版过松：

1. 麦克风：显示当前输入设备；绿色点表示已读取到设备。
2. 上屏：显示 Clipboard；绿色点表示上屏链路当前可用，失败态显示红点。
3. 按住说话模型：显示该模式当前默认模型，例如 `百度短语音` 或 `whisper.cpp`；绿色点表示所选模型配置完整。
4. 连续输入模型：显示该模式当前默认模型，例如 `百度短语音`、`whisper.cpp`，或 V8 预留的 `百度实时 WebSocket`；绿色点表示所选模型配置完整。

不再在准备状态里单独展示“本地识别”和“云端 API”。快捷键注册状态仍留在输入模式卡片里的小圆点，不进入准备状态。

## 模型选择页信息架构

模型选择页分成两个主要区域。

### 输入模式默认模型

顶部区域展示两行设置：

- 按住说话默认模型：`whisper.cpp`、`百度短语音`、`百度实时 WebSocket`。
- 连续输入默认模型：`whisper.cpp`、`百度短语音`、`百度实时 WebSocket`。

每个模型选项使用 segmented control 或紧凑按钮组。选中态表示该输入模式会使用这个模型；绿色小圆点表示该模型已配置可用；灰点表示未配置；禁用或实验态表示 V7 不能真实使用。

V7 默认值：

- 按住说话：`百度短语音`。
- 连续输入：`百度短语音`。

选择未配置模型时允许保存，但该模式在主界面显示未就绪，真实触发时应返回明确错误，不自动 fallback 到其他模型。

### 模型配置

下方区域按模型并列配置：

1. 本地 `whisper.cpp`
   - 可执行文件路径
   - 模型文件路径
   - 识别语言
   - 一键安装
   - 检测配置
   - 保存配置

2. 百度短语音识别
   - endpoint 固定为 `http://vop.baidu.com/server_api`
   - `BAIDU_ASR_API_KEY` 状态与密码输入保存
   - `BAIDU_ASR_SECRET_KEY` 状态与密码输入保存
   - `dev_pid`
   - `cuid`
   - `format`
   - `sample rate`
   - 可选 `lm_id`
   - 检测配置
   - 保存配置

3. 百度实时语音识别 WebSocket
   - endpoint 固定为 `wss://vop.baidu.com/realtime_asr`
   - `appkey` 来源说明：使用 `BAIDU_ASR_API_KEY`
   - `dev_pid`
   - `cuid`
   - `format=pcm`
   - `sample=16000`
   - 可选 `user`
   - 状态显示为 V8 预留，不参与 V7 真实转写

MiniMax 不出现在这三个配置入口中。

## 百度实时 WebSocket V8 边界

百度实时 WebSocket API 与短语音 API 不同。短语音 API 是录完后一次 HTTP 请求；WebSocket API 需要建立长连接、发送 START 帧、按约 160ms 发送 PCM 音频帧、处理服务端增量结果、发送结束帧、心跳、断线恢复和结果合并。

因此 V7 只保留配置和选择结构，不做真实调用。V8 再实现：

- `Ctrl+Alt+V` 连续输入模式优先使用 WebSocket。
- 音频采集过程按 16 kHz PCM 帧推送。
- 增量结果上屏和最终结果合并。
- 心跳、断线、取消、结束帧和错误恢复。

## 识别记录 metadata

每条识别记录新增 metadata：

- 输入方式：`按住说话` 或 `连续输入`。
- 使用模型：`whisper.cpp`、`百度短语音`、`百度实时 WebSocket`。
- 录音时长。
- 字数。

显示形式建议为正文下方一行小号信息，例如：

`按住说话 · 百度短语音 · 0:08 · 42 字`

连续输入仍保持“一次录音一条合并记录”，不把实时片段逐条写入历史。

## 识别准确率优化

V7 先设计两层确定性能力：

1. 百度 `lm_id`
   - 短语音标准版文档支持 `lm_id`，表示自训练平台模型 ID。
   - 只在百度短语音配置中作为可选字段保存。
   - 真实请求时如果配置了 `lm_id`，随短语音请求发送。

2. 本地术语纠错表
   - 维护一个用户可编辑的术语映射，例如 `scale -> skill`。
   - V7 可先实现数据结构和简单后处理函数，UI 可作为折叠配置区。
   - 后处理必须可关闭，并避免过度替换。

不在 V7 接入 LLM 纠错，避免隐私、延迟和不可控替换。

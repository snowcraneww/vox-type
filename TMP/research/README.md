# VoxType Research Workspace

This directory stores research materials for VoxType.

- `repos/`: local third-party repository clones for source reading. This directory is ignored by Git to avoid vendoring external projects.
- `requirements-brief.md`: structured understanding of the original VoxType requirement.
- `repo-candidates.md`: candidate repository inventory and screening notes.
- `solution-analysis.md`: comparative analysis and final technical recommendation.

Rules:

- Do not copy third-party source code into VoxType.
- Record source URLs and licenses.
- Keep facts separate from recommendations.
- Prefer linking to local file paths and upstream URLs over pasting large excerpts.
